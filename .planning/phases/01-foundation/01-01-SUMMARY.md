## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Breaking Change] Prisma 7 datasource URL not in schema.prisma**
- **Found during:** Task 2
- **Issue:** Prisma 7.8.0 removed `url = env("DATABASE_URL")` from schema.prisma. Config moved to `prisma.config.ts`. Running `npx prisma validate` with `url` in schema gives P1012 error.
- **Fix:** Removed `url` from schema.prisma `datasource` block; `prisma.config.ts` (auto-generated by `prisma init`) handles the connection URL.
- **Files modified:** prisma/schema.prisma

**2. [Rule 1 - Breaking Change] Prisma 7 PrismaClient requires driver adapter**
- **Found during:** Task 3
- **Issue:** Prisma 7 removed the built-in query engine for standard instantiation. `new PrismaClient({ log: [...] })` fails TypeScript — requires either `adapter` or `accelerateUrl`. For SQLite, the `@prisma/adapter-better-sqlite3` driver adapter is required.
- **Fix:** Installed `@prisma/adapter-better-sqlite3@7.8.0`; updated `src/lib/prisma.ts` to use `PrismaBetterSqlite3` adapter with `url` and `timeout` options. WAL pragma is applied automatically by the adapter from the DATABASE_URL query params.
- **Files modified:** src/lib/prisma.ts, package.json

**3. [Rule 1 - Breaking Change] Prisma 7 client generated locally, not @prisma/client**
- **Found during:** Task 3
- **Issue:** Prisma 7 generates the typed client to `src/generated/prisma/` (configurable via schema.prisma `output`). Import path is `@/generated/prisma/client`, not `@prisma/client`.
- **Fix:** Updated `src/lib/prisma.ts` import to `import { PrismaClient } from "@/generated/prisma/client"`.
- **Files modified:** src/lib/prisma.ts
